## 版本 1.6.5 - 开场白防剧透与触发时机修复

### 修改日期
2026-04-06

### 问题背景
- 开场白会在切换到“当前章节概览”时提前生成，导致与用户实际互动节奏不一致。
- 开场白提示词直接注入“本章摘要/目标/流程/关键节点”，存在剧情信息泄露风险。

### 本次改动
1. 触发时机收口（仅首章开幕与“下一章”触发）
- 文件：`txtToWorldbook/ui/chapterExperienceView.js`
- `showCurrentChapterPanelInternal` 不再自动调用开场白生成。
- 新增 `enterChapter(index, { triggerOpening })`：
   - 默认 `triggerOpening=true`，用于“开始阅读第一章”“下一章”。
   - 大纲页“查看当前章节概览”改为 `triggerOpening=false`，仅切换视图不触发生成。

2. 开场白提示词重写（50-100字，承上启下）
- 文件：`txtToWorldbook/ui/chapterExperienceView.js`
- `generateOpeningText` 改为“承上启下型开场白”提示词，硬性要求：
   - 仅输出 50-100 字中文；
   - 只做衔接与引入，不推进剧情；
   - 禁止泄露本章目标/流程/关键节点/转折/结局。
- 删除输入：本章摘要、本章目标、本章流程、本章关键节点。
- 新增输入：
   - 上一章标题/摘要/目标/流程/关键节点；
   - 最新一轮玩家输入与AI输出；
   - 本章原文开头 50-100 字片段。

3. 安全降级与长度约束
- 文件：`txtToWorldbook/ui/chapterExperienceView.js`
- 新增文本裁剪逻辑，超长输出自动收敛到 100 字以内。
- `buildOpeningFallback` 改为安全承接文案，不再使用本章摘要作为降级来源。
- 降级提示改为“已使用安全降级文案发送”。

4. 配置一致性
- 文件：`txtToWorldbook/core/constants.js`
- `chapterOpeningTargetLength` 默认值由 `120-220` 调整为 `50-100`。

### 验收要点
- 首章：点击“开始阅读第一章”后生成并发送开场白。
- 非首章：仅点击“下一章”进入时触发开场白。
- 切换到“当前章节概览”页不触发开场白生成。
- 开场白输出长度应在 50-100 字，且不泄露本章关键信息。

## 版本 1.6.1 - DS/中转站稳定化（TXT转世界书）

### 修改日期
2026-04-06

### 问题现象
- 在 OpenAI 兼容模式下，批量处理时出现大量 `400 Invalid max_tokens`。
- 并行处理中，参数类错误也会触发指数退避重试，导致日志刷屏。

### 根因分析
- OpenAI 兼容请求的 `max_tokens` 为硬编码 8192，在部分 DS/中转站网关上不兼容。
- 处理层未区分错误类型，4xx 参数错误与 429/5xx 一样会重试。

### 本次改动
1. 稳定默认值调整（偏向成功率）
- 默认分块字数从 15000 调整为 8000。
- 默认并发从 3 调整为 1。
- 新增 `customApiMaxTokens` 默认值 2048。

2. 设置面板与持久化
- 新增自定义 API 的 `Max Tokens` 输入项（建议 1024-4096）。
- 新增 `customApiMaxTokens` 保存与加载逻辑，并做 1-8192 安全夹取。
- 仅在 OpenAI 兼容提供商下显示该参数，减少误配置。

3. API 请求构建
- OpenAI 兼容请求的 `max_tokens` 改为读取 `customApiMaxTokens`（不再硬编码 8192）。
- 快速测试模型改为使用保守 token 上限（最高 1024），降低测试误报。

4. 重试策略降噪
- 新增可重试错误判定：
   - 允许重试：429、5xx、网络/超时类错误。
   - 不重试：400/401/403/404/422 等参数或鉴权类错误。
- 独立/串行流程和章节大纲生成流程均应用该策略，避免 4xx 连续重试刷屏。

### 涉及文件
- `txtToWorldbook/core/constants.js`
- `txtToWorldbook/ui/settingsPanel.js`
- `txtToWorldbook/ui/apiModeView.js`
- `txtToWorldbook/ui/eventBindings.js`
- `txtToWorldbook/services/settingsPersistenceService.js`
- `txtToWorldbook/services/apiService.js`
- `txtToWorldbook/services/processingService.js`

### 使用建议（DS/中转站）
- 推荐参数：`Max Tokens = 1024~4096`，`并发 = 1~2`，`每块字数 = 6000~9000`。
- 若仍偶发失败，优先降低并发到 1，再降低 `Max Tokens` 到 1536 或 1024。

每个memory(章节)新增字段：
- `chapterOutline` - 章节摘要
- `chapterScript` - 剧本(goal/flow/keyNodes)

## 版本 1.5.0 - 一期需求落地（去重/整理/token/角色分级）

### 修改日期
2026-04-06

### 版本说明
此版本对应 git tag `v1.5`，是一期需求的核心功能落地版本。

### 对应需求
1. 文件上传区域移动到面板最顶端（位于 API 设置之前）
2. 别名合并与重复条目处理重做（优先自动处理明显同名冲突）
3. 长章节处理时 token 控制优化（避免注入整本世界书）
4. 角色条目新增四级分类（主角/重要配角/普通配角/NPC）并贯通导出

### 关键实现

#### 1) UI 顺序调整
- 文件：`txtToWorldbook/ui/settingsPanel.js`
- 调整 `buildModalBodyHtml()` 的区块顺序：`文件上传` 提前到第一块。

#### 2) 别名合并重做
- 新增文件：`txtToWorldbook/services/nameNormalizationService.js`
   - 提供条目名归一化、显式同名判断、内容近重复检测、内容去重合并。
- 文件：`txtToWorldbook/services/mergeService.js`
   - 新增“明显同名冲突”自动合并能力（含 `_卷N` / 常见后缀归一化）。
   - 升级短名匹配和重复检测逻辑，降低漏检率。
   - 合并时不再简单拼接，改为去重合并内容并删除冲突条目。
- 文件：`txtToWorldbook/services/mergeWorkflowService.js`
   - 在 AI 两两判断前加入预处理：自动合并明显重复组。
   - 当预处理已消除重复时直接提示完成，减少无效 AI 调用。

#### 3) 整理条目重做（结构化去重）
- 文件：`txtToWorldbook/services/mergeWorkflowService.js`
   - `consolidateEntry` 增加结构化输出约束。
   - 新增本地 `dedupeStructuredContent`：对“字段: 值”内容进行字段级去重。
   - 整理完成提示修正为成功/失败分级提示。

#### 4) Token 控制优化
- 文件：`txtToWorldbook/services/processingService.js`
   - 串行处理路径移除“当前世界书全量 JSON 注入”。
   - 改为注入“相关世界书摘录（精简）”，显著降低 prompt 膨胀风险。
- 文件：`txtToWorldbook/services/worldbookService.js`
   - 增量合并内容由“前50字包含”升级为近重复检测 + 段落去重合并。

#### 5) 角色四级分类贯通
- 文件：`txtToWorldbook/core/constants.js`
   - 角色内容指引与系统提示词增加 `角色类型` 强制要求。
- 文件：`txtToWorldbook/services/parserService.js`
   - 解析后统一归一 `角色类型`，缺失时默认 `普通配角`。
- 文件：`txtToWorldbook/ui/renderer.js`
   - 世界书条目卡片展示角色类型徽标。
- 文件：`txtToWorldbook/services/exportFormatService.js`
   - 导出格式补充 `storyweaverRoleType` 元数据。
- 文件：`txtToWorldbook/services/importExportService.js`
   - 角色卡导出 `extensions.character_role_type`。
