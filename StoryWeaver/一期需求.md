# StoryWeaver 一期需求

## 痛点记录

### 1. 长周期角色信息累积问题
- 同一角色在多章节反复出现，信息不断追加
- 1000章后条目内容超过5000字，导致Token超限
- 早期关键设定被淹没在新内容底部

### 2. 分卷模式重复条目
- 开启分卷后产生 `李尘家别墅_卷1`、`李尘家别墅` 等重复条目
- 缺乏自动识别和合并机制

### 3. 条目演变无追踪
- 无法查看角色从第1章到第1000章的发展轨迹
- 角色属性变更（如25岁→26岁）被简单追加而非更新

### 4. 信息冗余
- AI每章输出的内容有大量重复描述
- 当前仅做简单字符串去重，效果有限
